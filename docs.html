Overview

Image to Strokes turns any photograph into an interactive 3D brushstroke painting. It runs entirely in your browser — no server, no uploads, no accounts.

How it works

The app runs a short analysis pipeline on your image, then uses the combined data to place and orient brushstrokes in 3D space. Each step feeds information to the stroke generator, so the result adapts to what the image actually contains.

Step 1
Upload & Analyze

Upload any photo. Five analyses run automatically: depth estimation, person segmentation, face landmarks, edge saliency, and color region detection. A status bar tracks progress.

Step 2
3D Painting

The 3D brushstroke result appears immediately beside the analysis panel. Orbit freely with mouse or touch. Each analysis thumbnail on the right shows what was detected — hover to enlarge.

Step 3
Adjust

Open the settings panel (tune icon) to adjust stroke count, size, color, render mode, and more. Changes rebuild the painting in real time. Toggle off any analysis that failed or isn't useful.

Step 4
Export

Save the canvas as PNG or JPG from the Canvas section in settings. The output image matches the canvas resolution you set, not the screen size.

Getting started

  1. 1
    Wait for model loading On first visit, three AI models are downloaded from public CDNs and cached locally. This takes 10–30 seconds depending on your connection. The status bar shows progress.
  2. 2
    Upload an image Click Upload image and choose any photo. Portraits give the richest results because face landmarks concentrate detail around the eyes, nose, and mouth. The app also works well with flowers, animals, and other colorful subjects.
  3. 3
    Review the analysis sidebar Six thumbnails appear on the right of the canvas as each analysis completes. Each shows what was detected — hover to enlarge. A status line under the title tells you whether the analysis succeeded and how it affects the painting. Use the ignore toggle to disable any result that looks wrong.
  4. 4
    Interact with the 3D painting Drag to orbit, scroll to zoom, right-drag or two-finger drag to pan. The canvas is live from page load — strokes appear as soon as analysis finishes.
  5. 5
    Tweak settings and export Open the settings panel with the tune icon. Adjust any parameter — the painting rebuilds automatically. When you are satisfied, save a PNG or JPG from the Canvas section.

Analysis sidebar

Six analysis thumbnails sit beside the 3D canvas. Each shows the raw output of one analysis pass. Hover a thumbnail to see a large preview. A status line under each title tells you whether the analysis succeeded and how it feeds into the painting. Use the ignore toggle to disable any result that looks wrong — the 3D view rebuilds immediately.

Original
Your uploaded image, displayed as-is. Every stroke samples its color directly from the original pixels at the stroke's start and end points, so this is the only analysis that is always active and cannot be ignored.
Segmentation
MediaPipe's selfie segmentation model classifies every pixel into one of six categories: background, hair, body skin, face skin, clothes, and others. Used in this demo: strokes are only seeded inside non-background pixels, so the painting fills the subject's silhouette rather than the whole frame. If the mask is judged unusable (less than 4% or more than 96% of the image is non-background), or if you toggle ignore, strokes spread across the whole image instead.
Depth Map
DepthAnything v2 (ONNX, fp32) estimates relative per-pixel depth. Brighter values are further from the camera; darker values are closer. Used in this demo: each stroke is placed on a 3D surface whose Z coordinate is read from this map, so the painting has genuine depth you can orbit around. Surface normals are also derived from the depth gradient, orienting strokes to lie tangentially on the reconstructed shape. Toggle ignore to flatten everything to a single plane.
Face Landmarks
MediaPipe's face landmarker detects 478 mesh points across the face. A curated subset of 24 key points is extracted — covering the eye corners, iris centres, nose bridge, nose tip, mouth corners, lip edges, and eyebrow arches. Used in this demo: stroke seeds are biased toward these feature points using a Gaussian probability field, so detail strokes cluster around the eyes, nose, and mouth while broader strokes fill the rest of the face. The Density ratio slider controls how strongly strokes cluster. If no face is detected, the app falls back to color region focus points automatically. Toggle ignore to remove the facial clustering entirely.
Edge Saliency
A Sobel gradient filter applied to a 256×256 luminance downsample of the image identifies regions with strong local contrast — contour edges, texture boundaries, and fine detail. The result is normalised to a 0–1 saliency map. Used in this demo: this map is visualised for inspection. It is computed without any AI model and always succeeds.
Subject Regions
HSV colour analysis on a 128×128 downsample finds the most saturated, non-green regions of the image. An 8×8 weighted-centroid grid produces up to 16 focus zones shown as yellow dots on the thumbnail. Used in this demo: when no face is detected these focus zones replace facial landmarks as the stroke-density attractor, pulling detail toward colourful subjects such as flowers, fabric, or animals. When segmentation is also unavailable, the subject probability map is used to filter stroke seeds so they concentrate in the most colourful areas. Toggle ignore to distribute strokes evenly regardless of colour.

3D Settings

Open the settings panel with the tune icon in the toolbar. All parameters rebuild the painting with a short debounce after you stop adjusting.

Strokes

Count 1 – 30 000 Total number of brushstrokes. Higher counts fill the image more densely but rebuild slower. 1 000–4 000 is a good starting range.
Density ratio 1 – 500 How strongly strokes are pulled toward feature zones (face landmarks or color focus points). At 1, strokes are distributed evenly across the subject. At 500, nearly all strokes cluster near the detected features.
Min length 2 – 20 Minimum number of steps in a stroke path. Strokes near feature zones (high detail) tend toward shorter lengths.
Max length 4 – 40 Maximum steps for strokes in low-detail areas. Longer strokes give a painterly, gestural feel away from focal regions.

Size & Layering

Large width 1 – 50 Width of strokes in low-detail areas. Higher values produce broad, impasto-style marks in the background.
Fine width 1 – 10 Width of strokes near feature points. Low values give delicate, hair-like marks around the eyes and other detail areas.
Detail sharpness 1 – 10 Controls how abruptly the width and length transition from coarse to fine as strokes approach feature points. Higher values make the focal zones feel more distinct.
Layer depth 0 – 20 How much detail-area strokes are pushed toward the camera in Z. Higher values make the focal region appear to float in front of the background layer.

Color

Hue rotation −180 – 180° Shifts all stroke colors around the color wheel. 0 preserves original colors; ±180 inverts the hue completely.
Saturation 0 – 200% Multiplier applied to saturation. Below 100 desaturates toward grey; above 100 intensifies colors. 90 is a slight mute that reads well as paint.
Color noise 0 – 100 Adds random variation to hue, saturation, and lightness per stroke, mimicking the unpredictability of physical paint mixing. 0 gives clean flat color; 70+ gives a lively, hand-painted feel.

Stroke Type

Legacy Ribbon geometry built from a single merged mesh. Fast to generate, uses a custom procedural brush texture baked into a canvas. All strokes share one draw call.
Reference Each stroke is rendered using the StrokeRenderer module — a higher-quality path that supports smoother cap shapes and richer noise modes. Slightly slower to build for very high stroke counts.

Stroke Style

Render mode
Flat Uniform color sampled from the stroke's start point.
Gradient along Color interpolates from start-point color to end-point color along the stroke's length.
Gradient across Color interpolates across the stroke's width.
Noise along Noise-warped gradient along the stroke length. Gives an organic, bristle-like color variation.
Noise across Noise-warped gradient across the stroke width.
Cap style
Square Stroke ends are cut flat — like a flat-bristle brush.
Rounded Ends are trimmed to a semicircle for a clean, marker-like feel.
Ragged Ends are broken up with noise, mimicking the dry-brush fraying of real bristles.

Surface

Surface jitter 0 – 30 Adds small random Z offsets to each stroke point, breaking up the perfectly smooth depth surface and giving a more textured feel.
Wireframe on / off Shows the underlying ribbon geometry as a white wireframe. Useful for understanding stroke topology. Only active in Legacy mode.
Cull front on / off Discards stroke faces whose surface normal points toward the camera. Reveals the back side of the depth surface when the view is rotated behind the subject.
Cull back on / off Discards stroke faces pointing away from the camera — the conventional behaviour for opaque objects.

Canvas

Width / Height 128 – 4096 Sets the actual WebGL canvas resolution in pixels. Larger sizes produce sharper exports but require more GPU memory. Click Apply to commit the change.
Background color picker Sets the scene background color. Use Pick on click to sample directly from the canvas, or Random sample to pull a random pixel from the original image.
Save PNG / JPG Exports the current canvas at its native resolution. PNG is lossless; JPG uses 95% quality and produces smaller files for sharing.

Save & Load

Analysis data can be saved between sessions when you run a local server. The Save and Load controls are hidden automatically when the app is served from a static host (GitHub Pages, etc.). 

# Install dependencies (Node.js required)
npm install

# Start the local server
./start.sh
# or: node server.js

Then open http://localhost:3000. Saved analyses are stored as JSON files in the analysis/ folder. Saving prompts for a name; loading lets you pick from the dropdown and press ↑.

On first launch with a local server, the app automatically loads the most recently saved analysis so you can continue where you left off.

Privacy

All processing runs in your browser. Your images and analysis results are never sent to any server. The AI models (DepthAnything v2 and the two MediaPipe models) are downloaded once from public CDNs (cdn.jsdelivr.net, storage.googleapis.com) and cached locally by the browser. On subsequent visits, they load from cache with no network request.
